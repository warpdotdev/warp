*Proposed change: Status line V2 (CODE-1890)*

*Summary:* Refactor the authenticated TUI footer in `crates/warp_tui/src/terminal_session_view.rs` from a split left-hint/right-metadata layout into one left-aligned status row with explicit sections and deterministic hint precedence.
*Key design choices:* Use a muted ` │ ` separator span; keep agent sections ordered as model, combined cwd/branch, usage, diff; make bash mode the first segment while omitting model and usage; make every footer hint (including the orchestration callout) replace the status row rather than mixing with metadata.
*Design alternatives:* Use `│` instead of ASCII `|` or the existing `•` because it matches the TUI's box-drawing borders and remains visually distinct from diff/count text. Remove the flex spacer instead of retaining separate left/right groups so every normal row starts at the same left edge. Combine cwd and branch into one section instead of separating them so the branch remains contextual to its path. Keep usage in agent mode, where it describes the selected conversation, but omit it in bash mode because it is stale AI-conversation metadata while the user is operating the shell. Preserve the existing hint priority (ctrl-c, loading, transient notice, then orchestration callout) and render the selected hint alone; shell mode is the exception because its indicator is a persistent status segment and must lead the bash row.
*Root cause / approach:* At commit `c86f67db3d61d5487301787e135ba890a7b9f243`, `render_footer` (`crates/warp_tui/src/terminal_session_view.rs:2066`) builds a left hint child, then puts the model/cwd/branch/usage/diff children in a second row after a flex child, which right-aligns the metadata. `render_left_footer_hint` (`terminal_session_view.rs:187`) also owns the obsolete `← for conversations` fallback. Build the footer as either (a) one lone hint row when a replacing hint is active, or (b) one `TuiFlex::row` containing only the selected status segments and muted ` │ ` separator spans. Reuse the existing `is_shell_mode`, `compact_footer_path`, git metadata, usage totals, and hint tone styles; do not change the separate focused-orchestration footer or the full-screen startup restore/error surfaces.
*Affected files:* `crates/warp_tui/src/terminal_session_view.rs`; `crates/warp_tui/src/terminal_session_view_tests.rs`.
*Open questions resolved:* (1) Separator is box-drawing `│`, rendered with one space on each side and muted styling. (2) Agent order is `[model] │ [cwd ↬ branch] │ [usage] │ [+N -M]`; cwd, usage, and diff remain conditional exactly as today, and cwd/branch are one segment. (3) Bash order is `[shell-mode] │ [cwd ↬ branch] │ [+N -M]`; model and usage are hidden. (4) Replacing states are the armed ctrl-c exit hint, conversation-list loading hint, and any `TransientHint` notice (muted or success); startup restore loading/failure still replaces the entire session view before the footer is rendered. Preserve the existing priority when multiple replacing states coexist. (5) `Shift + ↑ sub-agents` is a replacing navigation callout when orchestration tabs are available and the session is not in shell mode; shell mode wins so its first segment remains the shell indicator. The dedicated `render_orchestration_tab_footer` shown while tabs own focus is unchanged. The removed conversations callout never renders in any state.
*Risks / blast radius:* Footer layout, hint precedence, and bash/agent metadata visibility change; usage's clickable action and existing path/branch/diff formatting must remain intact in agent mode. The obsolete input-empty footer invalidation rationale must be removed, but the input subscription still must disarm ctrl-c confirmation, drive input detection, and notify rendering. The change is confined to one TUI view and its render-to-lines tests; no GUI, persistence, or protocol behavior changes.
