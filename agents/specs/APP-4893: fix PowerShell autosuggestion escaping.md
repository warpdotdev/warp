# APP-4893: fix PowerShell autosuggestion escaping

*Spec: PowerShell autosuggestions preserve Windows path separators*

== PRODUCT ==
*Summary:* On Windows sessions whose active shell is PowerShell, accepted autosuggestions must contain the command text the user would type directly: ordinary path separators remain single backslashes, while the two leading backslashes of a UNC path remain intact. The behavior applies to history, completer, and intelligent/AI next-command suggestions without changing the existing POSIX, WSL, or MSYS2 rules.

*Key design choices:* Normalize suggestion text at the client suggestion boundary using the active `ShellFamily`, before prefix stripping or editor insertion; apply the repair only to PowerShell semantics; preserve the leading UNC separator pair and leave POSIX-family suggestions untouched. Keep the editor's acceptance operation and the server API contract unchanged.

*Behavior* (numbered, testable invariants from the user's view):
1. In a native Windows PowerShell session, a suggestion for `cat \\WSL$\Ubuntu\home\dev\.intelligent-terminal\shell-integration_v2.sh` renders and accepts with exactly two leading UNC backslashes and one backslash between subsequent path components; no JSON-style doubled separators or backtick-prefixed separators are inserted.
2. The invariant holds whether the suggestion came from normal history recall, a completer path result, rich-history next-command prediction, or the server-backed AI next-command response, and whether the user accepts the full suggestion or a partial word/line.
3. A PowerShell command that already contains single separators is unchanged, and PowerShell metacharacters continue to use the existing backtick escaping behavior.
4. WSL and MSYS2 sessions retain their current path conversion and POSIX escaping behavior; a backslash in those shells is not normalized by the PowerShell repair.
5. POSIX shells (Bash, Zsh, and Fish) continue to use the existing backslash escaping rules for spaces and shell metacharacters.

== TECH ==
*Context:* `ShellFamily::PowerShell::escape` already omits `\` from `POWERSHELL_SHELL_ESCAPE_PATTERN` (`warpdotdev/warp/crates/warp_util/src/path.rs:23-45, 212-236 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`), and `SessionContext` correctly maps the active session to that family (`warpdotdev/warp/app/src/completer/mod.rs:294-350 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`). The defect is downstream: `Input::generate_autosuggestion_async` prioritizes partial next-command suggestions and also returns raw history/completer strings (`warpdotdev/warp/app/src/terminal/input.rs:9544-9637 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`); `NextCommandModel` passes rich-history and server responses through as complete commands (`warpdotdev/warp/app/src/ai/predict/next_command_model.rs:278-477 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`); `Input::on_autosuggestion_result` strips only the typed prefix (`warpdotdev/warp/app/src/terminal/input.rs:16313-16346 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`); and `EditorView::set_autosuggestion`/`insert_autosuggestion` store and insert the supplied text verbatim (`warpdotdev/warp/app/src/editor/view/mod.rs:3479-3518 and 5935-6040 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`). The AI request deliberately serializes history/context command records as JSON (`warpdotdev/warp/app/src/ai/predict/generate_ai_input_suggestions.rs:48-112, 179-199 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`); JSON serialization visibly doubles each backslash in the model-facing prompt, and the server asks the model to output a plain command without a client-side shell-escaping normalization (`warpdotdev/warp-server/router/handlers/generate_next_command.go:34-70 @ 0fb6c02dc158a8590c367860f9e5414313cf81d6`). A direct reproduction with a UNC command confirms the raw command decodes to single internal separators while the JSON prompt representation contains doubled separators.

*Design alternatives*:
- **Server-only prompt change:** Reformat or annotate JSON history so the model is less likely to copy JSON escapes. This may reduce occurrences but does not protect the client from already-escaped model output, older servers, or other suggestion producers; it is also outside the requested client repository.
- **Editor-only blind replacement:** Collapse every `\\` immediately before insertion. This is too late for prefix matching and can destroy the two leading backslashes required by UNC paths.
- **Selected — shell-aware client normalization before prefix handling:** Add one narrowly scoped normalizer for suggestion command text, keyed by `ShellFamily`, and invoke it before `strip_prefix`/editor insertion for both normal and intelligent suggestions. For PowerShell, collapse JSON-style duplicate separators in path tokens while preserving the leading UNC pair; make the operation idempotent. For POSIX/WSL/MSYS2, return the original text. This is deterministic, covers all current producers, and leaves the existing editor and server contracts intact.

*Proposed changes:* Add the normalizer in the client path/suggestion utility layer (reusing `ShellFamily` rather than OS detection), with explicit handling for token-start UNC prefixes and already-correct single separators. Apply it to the full suggestion result before `Input::on_autosuggestion_result` strips the typed buffer, and to the complete intelligent next-command value before `EditorView::maybe_populate_intelligent_autosuggestion` checks its prefix. Keep history/completer values that are already correct unchanged. Do not alter `ShellFamily::escape`, WSL/MSYS2 path conversion, or the server repository.

*Open questions resolved:* The reported path is a user-facing shell autosuggestion, not the passive prompt-chip MAA UI. The utility escape pattern and shell-family conversion are already correct, so the fix belongs at the suggestion text boundary. The two leading backslashes in `\\WSL$` are a valid UNC prefix and must be preserved; only redundant JSON-style duplicates after the token's UNC prefix are removed. No server change is required for the client fix, although the server prompt's JSON representation is the confirmed source that can feed pre-escaped AI output.

*Risks / blast radius:* A too-broad normalizer could alter literal PowerShell strings or UNC prefixes, and normalizing after prefix stripping could lose context about a UNC pair. Mitigate with a path-token-aware, idempotent implementation, tests for quoted/unquoted paths and typed-prefix splits, and a shell-family matrix. Applying the normalizer only at suggestion boundaries avoids changing command execution, clipboard drag/drop, or general path escaping.
