This document is based on local Git inspection of upstream `warpdotdev/warp` commits, public `gh pr view` and `gh issue view` metadata where it was available, current Warper source reads with line-number evidence, and the XP constraint that Warper implements only changes needed to keep the local-first product safe, non-corrupting, and buildable; its purpose is to decide which upstream changes are stop-ship enough to port, while separating upstream's reason from Warper's reason.

# Upstream Porting Suggestions
