Why the stable AVD was created
- The original Pixel_9_Pro_Fold AVD used a 16 GB RAM configuration and repeatedly hung before ADB became usable.
- Pixel_9_Pro_Fold_Stable was configured with lower RAM and cold-boot settings to avoid the earlier QEMU main-loop hang.
- The stable AVD booted successfully and reported sys.boot_completed=1.

Stable AVD config adjustments
- hw.ramSize=4096
- vm.heapSize=512
- hw.gpu.mode=swiftshader_indirect
- fastboot.forceFastBoot=no
- fastboot.forceColdBoot=yes

Launch the stable emulator
PowerShell command:
& "$env:LOCALAPPDATA\Android\Sdk\emulator\emulator.exe" -avd Pixel_9_Pro_Fold_Stable

Launch with a clean cold boot
PowerShell command:
& "$env:LOCALAPPDATA\Android\Sdk\emulator\emulator.exe" -avd Pixel_9_Pro_Fold_Stable -no-snapshot-load -no-snapshot-save -no-boot-anim

Verify boot
PowerShell commands:

& $adb -s emulator-5554 shell getprop sys.boot_completed

& $adb -s emulator-5554 shell getprop ro.build.version.release
& $adb -s emulator-5554 shell getprop ro.build.version.sdk

- ro.product.model=sdk_gphone64_x86_64

- ro.build.version.release=16
- ro.build.version.sdk=36

Shutdown emulator
PowerShell commands:
$adb = "$env:LOCALAPPDATA\Android\Sdk\platform-tools\adb.exe"
& $adb -s emulator-5554 emu kill

If graceful shutdown does not fully stop it:
Get-Process emulator, qemu-system-x86_64 -ErrorAction SilentlyContinue | Stop-Process -Force
& $adb kill-server
& $adb start-server
& $adb devices -l

Why repaired OTA images were not imported into Android SDK system-images
The repaired folders contain physical-device partition images such as:
- boot.img
- init_boot.img
- system.img
- vendor.img
- product.img
- vendor_boot.img
- vbmeta.img

They are missing Android Emulator SDK system-image package files:
- kernel-ranchu
- ramdisk.img
- source.properties
- package.xml

Because of that, copying these OTA images into C:\Users\Sarge\AppData\Local\Android\Sdk\system-images would create a broken or unrecognized emulator system image.

To create a true LineageOS emulator in the future
Use one of these sources:
1. A LineageOS emulator x86_64 SDK image package containing kernel-ranchu, ramdisk.img, system.img, vendor.img, source.properties, and package.xml.
2. A locally built LineageOS sdk_phone_x86_64 emulator image from the official LineageOS emulator build process.
